# 性能基准测试指南

## 测试环境要求

- 操作系统：Windows 11 Professional/Enterprise
- 硬件：至少 8GB RAM，4 核 CPU
- 网络：稳定的局域网环境

## 测试前准备

### 1. 构建 Release 版本

**PowerShell:**
```powershell
npm run tauri:build
```

**CMD:**
```cmd
npm run tauri:build
```

构建完成后，EXE 文件将自动复制到项目根目录，文件名格式：`prism-local-server-v3.0.0.exe`

### 2. 准备测试资源

在测试目录中准备以下文件：
- `index.html` - 简单的 HTML 页面（< 10KB）
- `style.css` - CSS 样式文件（< 50KB）
- `script.js` - JavaScript 文件（< 100KB）
- `test-image.png` - 图片文件（约 500KB）
- `test-video.mp4` - 视频文件（约 50MB，用于测试 Range Requests）

## 性能优化建议

如果某些指标未达标，可以尝试以下优化：

### 启动时间优化
- 减少应用启动时的初始化操作
- 延迟加载非关键模块
- 优化 Rust 依赖项编译

### 内存占用优化
- 检查是否有内存泄漏
- 优化 Tokio 运行时配置
- 减少不必要的数据缓存

### 文件体积优化
- 进一步优化 Cargo.toml 的 release profile
- 移除未使用的依赖项
- 使用 UPX 压缩 EXE（可选）

### 响应时间优化
- 优化 IPC 调用链路
- 减少不必要的状态更新
- 使用 React.memo 和 useMemo 优化渲染
