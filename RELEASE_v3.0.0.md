# Prism Local Server v3.0.0 发布说明

## 🎉 重大更新：完整 Tauri v2 重写

这是 Prism Local Server 的全新版本，从 Python/Flet 架构完全迁移到 Tauri v2 (Rust + React)，带来了革命性的性能提升和现代化的用户体验。

---

## ✨ 核心特性

### 🚀 性能飞跃
- **内存占用**：空闲状态 ≤ 50MB（相比旧版本降低 70%）
- **启动速度**：冷启动 ≤ 1.5 秒（原生 Rust 编译优势）
- **包体积**：EXE 文件仅 4.72 MB（相比 Electron 方案减少 80%）
- **并发能力**：基于 Tokio 异步运行时，轻松支持 10+ 服务实例

### 💎 Windows 11 Fluent Design UI
- **毛玻璃效果**：Mica/Acrylic 背景，完美融入系统主题
- **流畅动画**：Framer Motion 驱动的丝滑交互体验
- **深色模式**：默认启用，符合 Windows 11 设计语言
- **精致细节**：12px 圆角、分层阴影、Hover 微动效

### 🎯 智能服务管理
- **智能命名**：自动使用部署目录名称作为服务名（支持中文）
- **端口管理**：自动检测端口占用并递增，智能提示最终使用端口
- **唯一性约束**：同一目录只能部署一次，避免冲突
- **多实例数据共享**：多窗口间实时同步服务列表

### 🔧 技术架构
- **后端**：Rust + Axum（高性能 HTTP 静态文件服务器）
- **前端**：React 18 + TypeScript + Tailwind CSS + shadcn/ui
- **构建工具**：Vite + Tauri CLI
- **运行时**：原生二进制 + WebView2（系统自带）

### 📦 功能完善
- ✅ 支持 HTTP Range Requests（视频拖拽播放、断点续传）
- ✅ 自动识别 MIME Types（CSS/JS/图片/视频等）
- ✅ 局域网访问支持（自动显示本机 IP）
- ✅ 系统托盘集成（后台运行、快速控制）
- ✅ 浏览器自动打开（ACL 权限已修复）
- ✅ 默认部署目录智能识别（EXE 所在位置）

## 📥 下载与安装

### 安装包选择

| 文件名 | 类型 | 大小 | 说明 |
|--------|------|------|------|
| `prism-local-server-v3.0.0.exe` | 便携版 | 4.72 MB | 无需安装，双击即用 |
| `Prism Local Server_3.0.0_x64_en-US.msi` | MSI 安装包 | 2.43 MB | Windows Installer 标准安装 |
| `Prism Local Server_3.0.0_x64-setup.exe` | NSIS 安装包 | 1.74 MB | 自定义安装向导 |

### 推荐安装方式

**首次使用推荐**：下载 `prism-local-server-v3.0.0.exe` 便携版
- ✅ 无需安装，解压即用
- ✅ 可放置在任意目录
- ✅ 默认部署当前目录的 HTML 文件

**企业部署推荐**：使用 MSI 安装包
- ✅ 支持静默安装（`msiexec /i "Prism Local Server_3.0.0_x64_en-US.msi" /quiet`）
- ✅ 自动添加到开始菜单
- ✅ 支持组策略管理

## 🔄 从旧版本迁移

### 配置文件兼容性
v3.0.0 使用全新的配置格式，旧版本配置文件不兼容。首次启动时会自动创建新配置文件。

### 数据迁移
无需迁移数据，v3.0.0 不保存服务历史记录（每次启动为全新状态）。

## 📋 系统要求

- **操作系统**：Windows 11（推荐）或 Windows 10 1809+
- **运行时**：WebView2（Windows 11 自带，Win10 需安装）
- **内存**：最低 100 MB 可用内存
- **磁盘**：最低 20 MB 可用空间

## 🐛 已知问题

1. **WebView2 依赖**：Windows 10 用户首次运行可能需要安装 WebView2 Runtime
2. **防火墙提示**：首次启动服务时 Windows 防火墙可能弹出提示，请允许访问

## 🛠️ 快速开始

### 1. 启动应用
双击 `prism-local-server-v3.0.0.exe`

### 2. 部署网站
- 在"部署目录"输入框中选择包含 HTML 文件的文件夹
- 点击"启动服务"按钮
- 应用会自动打开浏览器访问部署的网站

### 3. 局域网访问
- 服务卡片中会显示"局域网访问"地址
- 其他设备可通过该地址访问（需确保防火墙允许）
