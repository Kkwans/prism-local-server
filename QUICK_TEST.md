# 快速测试指南

## 🚀 立即测试应用

### 1. 运行应用

双击运行：
```
release\Prism-Local-Server-v3.0.0.exe
```

### 2. 基础功能测试（5 分钟）

#### 测试 1：启动服务
1. 点击"浏览"按钮，选择 `tests/test_resources` 目录
2. 点击"启动服务"按钮
3. ✅ 检查：浏览器是否自动打开
4. ✅ 检查：页面是否正确显示（包含标题、样式、图片）
5. ✅ 检查：Toast 提示是否显示

#### 测试 2：端口自动递增
1. 打开命令行，运行：`python -m http.server 8888`（占用 8888 端口）
2. 在应用中再次启动服务（使用相同目录）
3. ✅ 检查：是否显示"端口已自动调整"提示
4. ✅ 检查：服务是否在 8889 端口启动

#### 测试 3：多服务并发
1. 启动 3-5 个不同目录的服务
2. ✅ 检查：所有服务是否都正常运行
3. ✅ 检查：每个服务卡片是否显示正确信息
4. ✅ 检查：运行时长是否实时更新

#### 测试 4：系统托盘
1. 点击窗口关闭按钮（X）
2. ✅ 检查：窗口是否隐藏（而非退出）
3. ✅ 检查：系统托盘是否显示图标
4. 右键点击托盘图标
5. ✅ 检查：是否显示菜单（显示主窗口、退出程序）
6. 点击"显示主窗口"
7. ✅ 检查：窗口是否重新显示

#### 测试 5：停止和重启服务
1. 点击某个服务的"停止"按钮
2. ✅ 检查：服务是否停止
3. ✅ 检查：浏览器访问是否失败
4. 点击"重启"按钮
5. ✅ 检查：服务是否重新启动

### 3. UI 效果测试（2 分钟）

#### Mica 毛玻璃效果
1. 确保使用 Windows 11 系统
2. 将应用窗口移动到不同背景上
3. ✅ 检查：窗口背景是否有毛玻璃模糊效果
4. ✅ 检查：背景是否随着后面内容变化

#### 动画效果
1. 启动/停止服务
2. ✅ 检查：服务卡片是否有进入/退出动画
3. 鼠标悬停在按钮上
4. ✅ 检查：按钮是否有 Hover 缩放效果

### 4. 性能快速检查（3 分钟）

#### 启动速度
1. 完全关闭应用
2. 使用秒表计时
3. 双击运行应用
4. ✅ 检查：从点击到窗口显示是否 ≤ 1.5 秒

#### 内存占用
1. 打开任务管理器（Ctrl+Shift+Esc）
2. 找到"Prism Local Server"进程
3. ✅ 检查：空闲状态内存是否 ≤ 40MB
4. 启动 1 个服务
5. ✅ 检查：内存是否 ≤ 80MB
6. 启动 5 个服务
7. ✅ 检查：内存是否 ≤ 150MB

#### 文件加载速度
1. 在浏览器中打开服务页面
2. 打开开发者工具（F12）→ Network 标签
3. 刷新页面
4. ✅ 检查：CSS/JS/图片加载时间是否 < 10ms

### 5. 局域网访问测试（可选，5 分钟）

1. 启动一个服务
2. 复制服务卡片上显示的局域网地址（如 `http://192.168.1.100:8888/index.html`）
3. 在手机浏览器中打开该地址
4. ✅ 检查：页面是否正常显示
5. ✅ 检查：所有资源是否正确加载

### 6. 视频拖拽测试（可选，需要视频文件）

1. 在测试目录中放置一个 MP4 视频文件
2. 创建一个 HTML 文件引用该视频：
```html
<!DOCTYPE html>
<html>
<body>
  <video controls width="800">
    <source src="test.mp4" type="video/mp4">
  </video>
</body>
</html>
```
3. 启动服务并打开该 HTML
4. ✅ 检查：视频是否能播放
5. ✅ 检查：拖拽进度条是否能快速跳转（Range Request）

## 🐛 如果发现问题

### 应用无法启动
- 检查是否安装了 WebView2 运行时
- 检查系统是否为 Windows 10 1809+ 或 Windows 11
- 查看日志文件（如果有）

### 端口被占用
- 检查是否有其他程序占用了 8888-8900 端口
- 尝试手动指定更高的端口号（如 9000）

### 浏览器未自动打开
- 检查默认浏览器设置
- 手动复制地址到浏览器打开

### Mica 效果不显示
- 确认系统为 Windows 11
- 确认系统设置中启用了透明效果
- 重启应用

## ✅ 测试通过标准

如果以上测试都通过，说明应用已经可以正常使用！
