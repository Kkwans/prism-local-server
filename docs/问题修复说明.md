# 问题修复说明

## 修复时间
2026-03-16

## 问题描述

### 问题1: Flet 0.24+ 版本 colors 模块导入错误

**错误信息:**
```
The application encountered an error: module 'flet' has no attribute 'colors'
```

**根本原因:**
Flet 0.24+ 版本中，`colors` 模块需要单独导入，不能直接通过 `ft.colors` 访问。

**影响范围:**
- `main.py`
- `ui/home_view.py`
- `ui/settings_dialog.py`

### 问题2: build.bat 中文乱码

**错误表现:**
运行 `build.bat` 时，终端输出的中文显示为乱码。

**根本原因:**
Windows CMD 默认使用 GBK 编码，而批处理文件使用 UTF-8 编码。

## 修复方案

### 修复1: colors 模块导入

**修改内容:**

1. **main.py**
   ```python
   # 添加导入
   from flet import colors
   
   # 修改使用
   color_scheme_seed=colors.BLUE  # 原: ft.colors.BLUE
   ```

2. **ui/home_view.py**
   ```python
   # 添加导入
   from flet import colors
   
   # 批量替换所有 ft.colors.XXX 为 colors.XXX
   # 共修复约 40 处
   ```

3. **ui/settings_dialog.py**
   ```python
   # 添加导入
   from flet import colors
   
   # 修复 2 处使用
   ```

### 修复2: 批处理文件编码

**修改内容:**

在所有 `.bat` 文件的第二行添加：
```batch
chcp 65001 >nul
```

这会强制 CMD 使用 UTF-8 编码，解决中文乱码问题。

**影响文件:**
- `build.bat`
- `pack.bat`
- `run.bat`

## 打包说明

### 推荐使用 Flet 官方打包工具

**命令:**
```bash
pack.bat
```

**打包流程:**
1. 清理旧的构建文件
2. 使用 `flet pack` 命令打包
3. 当询问 "Do you want to delete 'build' directory? (y/n)" 时，输入 `y`
4. 等待打包完成（可能需要几分钟）
5. 生成的 EXE 文件位于 `dist/PrismLocalServer.exe`

### 不推荐使用 PyInstaller

**原因:**
- PyInstaller 对 Flet 框架支持不完善
- 缺少 `flet_desktop` 模块
- 打包后的 EXE 无法正常运行
