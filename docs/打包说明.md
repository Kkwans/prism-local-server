# 打包说明

## 推荐方法：使用 Flet 官方打包工具

### 为什么使用 Flet 官方打包？

Flet 是基于 Flutter 的框架，需要特殊的运行时支持。使用 Flet 官方的 `flet pack` 命令可以：
- ✅ 自动包含所有必需的 Flet 运行时组件
- ✅ 正确处理 flet_desktop 模块
- ✅ 生成可靠的独立 EXE 文件
- ✅ 避免 PyInstaller 的兼容性问题

### 打包步骤

#### 1. 安装依赖
```bash
pip install -r requirements.txt
```

#### 2. 生成图标（可选）
```bash
python create_modern_icon.py
```
这会在 `assets/` 目录生成：
- `icon.png` - 用于 Flet 打包
- `icon.ico` - Windows 图标格式

#### 3. 运行打包脚本
```bash
pack.bat
```

或者手动运行：
```bash
flet pack main.py --name "PrismLocalServer" --icon assets/icon.png --add-data "assets;assets"
```

#### 4. 查找生成的 EXE
打包完成后，EXE 文件位于：
```
dist/PrismLocalServer.exe
```

### 打包参数说明

- `--name "PrismLocalServer"` - 指定 EXE 文件名
- `--icon assets/icon.png` - 指定应用图标
- `--add-data "assets;assets"` - 包含资源文件夹

### 常见问题

#### Q1: 打包失败，提示 "flet: command not found"
**解决方案**：
```bash
pip install --upgrade flet
```

#### Q2: 首次运行 EXE 很慢
**原因**：Flet 首次运行时会下载运行时组件（约 50MB）

**解决方案**：耐心等待，后续运行会很快

#### Q3: 打包后的 EXE 很大（100MB+）
**原因**：Flet 包含了完整的 Flutter 运行时

**说明**：这是正常的，Flet 应用的体积通常在 80-150MB

#### Q4: 想要更小的 EXE 体积
**建议**：
1. 使用 UPX 压缩（可能不稳定）
2. 考虑使用安装包而不是单文件 EXE
3. 接受 Flet 的体积特性（换取现代化 UI）

### 备用方法：PyInstaller（不推荐）

如果必须使用 PyInstaller，请注意：
- ⚠️ 可能遇到 flet_desktop 模块问题
- ⚠️ 需要手动配置大量隐藏导入
- ⚠️ 打包后可能无法运行

**不推荐使用 build.bat（PyInstaller 方式）**
