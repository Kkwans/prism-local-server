## 快速开始

### 第一次使用

1. **启动程序**
   - 双击 `PrismLocalServer.exe` 启动程序
   - 程序会自动创建配置文件和日志目录

2. **选择部署目录**
   - 点击"📁 选择目录"按钮
   - 选择包含HTML文件的目录
   - 或者直接使用默认目录（程序所在目录）

3. **启动服务**
   - 点击"🚀 启动服务"按钮
   - 程序会自动检测HTML文件
   - 浏览器自动打开访问地址

4. **访问网页**
   - 本地访问：`http://localhost:8888/index.html`
   - 局域网访问：`http://你的IP:8888/index.html`
   - 手机/平板访问：使用局域网地址

### 5分钟快速教程

```
1. 准备HTML文件
   └─ 我的网页/
      ├─ index.html
      ├─ css/
      │  └─ style.css
      ├─ js/
      │  └─ script.js
      └─ images/
         └─ logo.png

2. 启动Prism Local Server

3. 选择"我的网页"目录

4. 点击"启动服务"

5. 完成！浏览器自动打开网页
```

## 功能说明

### 1. 启动服务

**操作步骤：**
1. 选择部署目录（可选）
2. 点击"启动服务"按钮
3. 等待服务启动（通常1-2秒）
4. 浏览器自动打开

**自动功能：**
- 自动检测HTML文件（index.html或messages.html）
- 自动查找可用端口（8888被占用时自动递增）
- 自动打开浏览器（可在配置中关闭）
- 自动生成局域网访问地址

**注意事项：**
- 确保目录中有HTML文件
- 确保端口未被其他程序占用
- 防火墙可能需要允许访问

### 2. 停止服务

**操作步骤：**
1. 在服务列表中找到要停止的服务
2. 点击"⏹️ 停止"按钮
3. 确认停止操作

**效果：**
- 立即停止HTTP服务
- 释放端口资源
- 从服务列表中移除

### 3. 多服务管理

**功能说明：**
- 支持同时运行多个服务
- 每个服务使用不同端口
- 独立管理，互不影响

**使用场景：**
- 同时开发多个项目
- 对比不同版本
- 团队协作演示

**操作方法：**
1. 启动第一个服务（端口8888）
2. 选择另一个目录
3. 再次点击"启动服务"（自动使用端口8889）
4. 重复以上步骤可启动更多服务

### 4. 主题切换

**支持主题：**
- 🌙 浅色主题（Light）
- ☀️ 深色主题（Dark）
- 🌓 系统主题（System）

**切换方法：**
- 点击主题按钮（🌙图标）
- 循环切换：浅色 → 深色 → 系统 → 浅色

**自动跟随：**
- 系统主题模式会自动跟随Windows系统主题
- Windows切换主题时，应用自动更新

### 5. 后台运行

**功能说明：**
- 关闭主窗口后，服务继续运行
- 通过系统托盘管理
- 最小化资源占用

**操作方法：**
1. 点击窗口关闭按钮（X）
2. 选择"是 - 最小化到托盘"
3. 程序隐藏到系统托盘

**恢复显示：**
- 双击托盘图标
- 或右键选择"显示/隐藏"

### 6. 配置管理

**可配置项：**
- 默认端口（1024-65535）
- 默认目录
- 入口HTML文件
- 自动打开浏览器

**配置保存：**
- 配置自动保存到 `config/settings.json`
- 下次启动自动加载
- 支持恢复默认配置

## 常见问题

### Q1: 启动服务失败，提示"端口被占用"

**原因：**
- 端口8888已被其他程序使用

**解决方法：**
1. 方法一：程序会自动尝试下一个端口（8889）
2. 方法二：在配置中修改默认端口
3. 方法三：关闭占用端口的程序

**检查端口占用：**
```bash
# 在命令提示符中执行
netstat -ano | findstr :8888
```

### Q2: 局域网无法访问

**可能原因：**
1. 防火墙阻止
2. 网络未连接
3. IP地址错误

**解决方法：**
1. 允许防火墙访问：
   - 打开Windows防火墙设置
   - 允许Prism Local Server通过防火墙
   
2. 检查网络连接：
   - 确保电脑和手机在同一局域网
   - 检查WiFi连接状态

3. 确认IP地址：
   - 使用程序显示的局域网地址
   - 或在命令提示符执行 `ipconfig` 查看IP

### Q3: CSS/JS文件加载失败

**原因：**
- HTML中使用了绝对路径
- 文件路径大小写不匹配

**解决方法：**
1. 使用相对路径：
   ```html
   <!-- 正确 -->
   <link href="css/style.css" rel="stylesheet">
   
   <!-- 错误 -->
   <link href="/css/style.css" rel="stylesheet">
   <link href="C:/MyWeb/css/style.css" rel="stylesheet">
   ```

2. 检查文件名大小写：
   - Windows不区分大小写
   - 但HTTP服务器区分大小写
   - 确保HTML中的路径与实际文件名一致

### Q4: 视频无法播放

**可能原因：**
1. 视频格式不支持
2. 浏览器不支持
3. 文件路径错误

**解决方法：**
1. 使用常见格式：MP4、WebM
2. 使用现代浏览器：Chrome、Edge、Firefox
3. 检查视频文件路径

### Q5: 程序无法启动

**可能原因：**
1. 缺少依赖文件
2. 权限不足
3. 系统不兼容

**解决方法：**
1. 确保所有文件完整（不要单独复制EXE）
2. 以管理员身份运行
3. 确认系统为Windows 11

### Q6: 内存占用过高

**正常范围：**
- 空闲状态：50-80MB
- 单服务运行：60-100MB
- 多服务运行：80-150MB

**优化方法：**
1. 关闭不需要的服务
2. 减少同时运行的服务数量
3. 重启程序释放内存

## 技巧与建议

### 技巧1: 快速部署

**方法：**
1. 将 `PrismLocalServer.exe` 复制到项目目录
2. 双击运行
3. 直接点击"启动服务"（自动使用当前目录）

### 技巧2: 团队协作

**场景：**
团队成员需要查看你的网页

**步骤：**
1. 启动服务
2. 查看局域网地址（如 `http://192.168.1.100:8888`）
3. 分享地址给团队成员
4. 团队成员在浏览器中打开

### 技巧3: 移动设备测试

**场景：**
测试网页在手机/平板上的显示效果

**步骤：**
1. 确保电脑和手机在同一WiFi
2. 启动服务，获取局域网地址
3. 在手机浏览器中输入地址
4. 实时查看效果

**提示：**
- 可以生成二维码方便扫描
- 支持多设备同时访问

### 技巧4: 开发调试

**场景：**
前端开发时需要频繁刷新

**建议：**
1. 保持服务运行
2. 修改代码后直接刷新浏览器
3. 无需重启服务

**注意：**
- 静态文件修改立即生效
- 无需重新部署

### 技巧5: 性能优化

**建议：**
1. 避免部署过大的目录（>10GB）
2. 将大文件放在子目录中
3. 使用压缩后的资源文件
4. 定期清理日志文件

### 技巧6: 安全建议

**注意事项：**
1. 不要在公网暴露服务
2. 仅在可信网络中使用
3. 不要部署敏感信息
4. 定期更新程序版本

### 端口说明

**默认端口：** 8888

**端口范围：** 1024-65535

**自动递增：** 8888 → 8889 → 8890 → ...

**保留端口：** 避免使用1-1023（系统保留）

### 系统要求

**操作系统：**
- Windows 11 (22H2或更高版本)

**硬件要求：**
- CPU: 1GHz或更快
- 内存: 2GB或更多
- 磁盘: 100MB可用空间

**网络要求：**
- 局域网访问需要网络连接
- 本地访问无需网络
