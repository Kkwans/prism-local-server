## 快速开始

### 第一次使用

1. **启动应用**
   - 双击桌面图标或从开始菜单启动
   - 应用会在 1-2 秒内完成启动

2. **部署你的第一个网站**
   - 在"启动新服务"区域，点击"浏览"按钮
   - 选择包含 HTML 文件的目录（如 `C:\MyWebsite`）
   - 端口号保持默认 8888（或自定义）
   - 点击"启动服务"按钮
   - 浏览器会自动打开并显示你的网页

3. **分享给局域网设备**
   - 在服务卡片中找到"局域网访问"地址
   - 在手机/平板浏览器中输入该地址
   - 即可访问你的网页

### 服务卡片说明

每个运行中的服务都会显示一个卡片，包含：

- **服务名称**: 自动使用目录的最后一级文件夹名
- **端口号**: 服务监听的端口
- **目录路径**: 部署的目录完整路径
- **本地访问地址**: 在本机浏览器中访问的 URL
- **局域网访问地址**: 在局域网其他设备中访问的 URL
- **运行时长**: 服务已运行的时间（实时更新）
- **操作按钮**:
  - 打开浏览器：在默认浏览器中打开服务
  - 复制链接：复制本地访问地址到剪贴板
  - 重启：重启服务（保持端口和配置不变）
  - 停止：停止服务并释放端口

## 功能详解

### 1. 启动服务

#### 基本步骤
1. 点击"浏览"按钮选择目录
2. 设置端口号（可选，默认 8888）
3. 点击"启动服务"按钮

#### 智能特性

**自动端口递增**:
- 如果指定端口被占用，应用会自动尝试下一个端口
- 最多尝试 100 次（8888 → 8988）
- 成功后显示 Toast 提示："原端口 8888 被占用，已自动切换到端口 8889"

**智能服务命名**:
- 服务名称自动使用目录的最后一级文件夹名
- 支持中文、日期、下划线等特殊字符
- 示例：`C:\Users\Kkwans\Documents\我的收藏_2026-01-20` → 服务名称为 "我的收藏_2026-01-20"
- 长名称会自动截断到 50 字符并添加省略号

**目录唯一性保护**:
- 同一目录只能启动一个服务
- 如果尝试重复启动，会显示错误："目录已被服务 'xxx' 使用（端口 8888）"
- 必须先停止已有服务才能重新部署该目录

### 2. 管理服务

#### 停止服务
1. 点击服务卡片上的"停止"按钮
2. 确认对话框中点击"确认停止"
3. 服务立即停止，端口和目录被释放
4. 服务卡片从列表中移除

#### 重启服务
1. 点击服务卡片上的"重启"按钮
2. 服务会先停止，然后使用相同配置重新启动
3. 端口号保持不变
4. 适用于更新了网页文件后需要刷新的场景

#### 复制链接
1. 点击"复制链接"按钮
2. 本地访问地址自动复制到剪贴板
3. 可以粘贴到浏览器或分享给他人

#### 在浏览器中打开
1. 点击"打开浏览器"按钮
2. 使用系统默认浏览器打开服务地址
3. 也可以直接点击服务卡片中的 URL 链接

### 3. 配置设置

点击右上角"设置"按钮打开设置对话框。

#### 可配置项

**默认端口号**:
- 启动新服务时的默认端口
- 范围：1024-65535
- 建议：8888-9999

**默认部署目录**:
- 启动新服务时的默认目录
- 首次启动时自动识别为 EXE 所在目录
- 可以手动修改为常用目录
- 修改后会持久化保存

**默认入口文件**:
- 服务的入口 HTML 文件名
- 默认：index.html
- 可以修改为 home.html、main.html 等

**自动打开浏览器**:
- 启动服务后是否自动打开浏览器
- 默认：开启
- 关闭后需要手动点击"打开浏览器"按钮

**最小化到托盘**:
- 关闭窗口时是否最小化到系统托盘
- 默认：开启
- 关闭后，点击窗口关闭按钮会直接退出程序

#### 保存设置
1. 修改配置后点击"保存"按钮
2. 配置立即生效并持久化保存
3. 主界面的输入框会自动更新为新的默认值

### 4. 局域网访问

#### 获取局域网地址
1. 启动服务后，在服务卡片中找到"局域网访问"部分
2. 地址格式：`http://[局域网IP]:[端口]`
3. 示例：`http://192.168.1.100:8888`

#### 在其他设备上访问
1. 确保设备连接到同一局域网（相同 WiFi 或有线网络）
2. 在设备浏览器中输入局域网地址
3. 即可访问部署的网页

#### 防火墙设置
如果局域网无法访问，需要配置防火墙：

**方法 1: 允许应用通过防火墙**
1. 打开 Windows 设置 → 隐私和安全性 → Windows 安全中心
2. 点击"防火墙和网络保护"
3. 点击"允许应用通过防火墙"
4. 找到"Prism Local Server"并勾选"专用"和"公用"

**方法 2: 添加端口规则**
1. 打开 Windows Defender 防火墙 → 高级设置
2. 点击"入站规则" → "新建规则"
3. 选择"端口" → TCP → 特定本地端口 → 输入 8888-8988
4. 选择"允许连接" → 完成

### 5. 视频播放

#### 支持的视频格式
- MP4 (H.264/H.265)
- WebM (VP8/VP9)
- MOV (QuickTime)
- 其他浏览器支持的格式

#### Range Request 支持
本工具完整实现了 HTTP Range Request 协议，支持：
- ✅ 视频拖拽播放（无需重新加载）
- ✅ 断点续传（网络中断后继续下载）
- ✅ 分段加载（节省带宽）

#### 测试 Range Request
1. 在浏览器中打开视频文件
2. 打开开发者工具（F12） → Network 标签
3. 拖拽视频进度条
4. 观察网络请求：
   - 请求头包含 `Range: bytes=xxx-`
   - 响应状态码为 `206 Partial Content`
   - 响应头包含 `Content-Range: bytes xxx-yyy/zzz`

## 高级用法

### 1. 部署 SPA 应用

对于 React、Vue、Angular 等单页应用：

1. 构建生产版本：
   ```bash
   npm run build
   ```

2. 在应用中选择 `dist` 或 `build` 目录

3. 启动服务

4. 访问 `http://localhost:8888`

### 2. 部署静态博客

对于 Hexo、Hugo、Jekyll 等静态博客：

1. 生成静态文件：
   ```bash
   hexo generate  # Hexo
   hugo           # Hugo
   ```

2. 在应用中选择 `public` 目录

3. 启动服务

### 3. 测试移动端适配

1. 在电脑上启动服务
2. 在手机浏览器中访问局域网地址
3. 测试移动端布局和交互

### 4. 多项目并行开发

1. 同时启动多个服务（不同目录）
2. 每个项目使用不同端口
3. 在浏览器中打开多个标签页
4. 实时预览多个项目

### 5. 自定义入口文件

如果你的项目入口文件不是 `index.html`：

1. 打开设置对话框
2. 修改"默认入口文件"为你的文件名（如 `home.html`）
3. 保存设置
4. 启动服务时会自动使用该文件

## 常见问题

### Q1: 如何更改服务的端口号？
A: 需要先停止服务，然后重新启动并指定新的端口号。运行中的服务无法直接修改端口。

### Q2: 可以同时运行多少个服务？
A: 理论上无限制，但建议不超过 10 个以保证性能。每个服务约占用 10-20MB 内存。

### Q3: 服务会在电脑重启后自动启动吗？
A: 不会。需要手动启动应用并重新启动服务。未来版本可能会添加此功能。

### Q4: 如何完全退出程序？
A: 右键系统托盘图标 → 选择"退出"。直接关闭窗口只会最小化到托盘。

### Q5: 配置文件保存在哪里？
A: 配置文件保存在：
```
C:\Users\[用户名]\AppData\Roaming\prism-local-server\config.json
```

### Q6: 如何重置所有设置？
A: 删除配置文件（见 Q5），然后重启应用。应用会自动创建默认配置。

### Q7: 支持 HTTPS 吗？
A: 当前版本仅支持 HTTP。HTTPS 支持计划在未来版本中添加。

### Q8: 可以自定义 HTTP 响应头吗？
A: 当前版本不支持。这是一个计划中的扩展功能。

### Q9: 如何查看服务的访问日志？
A: 当前版本不支持访问日志。这是一个计划中的扩展功能。

### Q10: 服务名称可以自定义吗？
A: 当前版本服务名称自动使用目录名。未来版本可能支持自定义。

## 性能优化建议

### 减少内存占用
1. 关闭不需要的服务实例
2. 避免部署包含大量文件的目录
3. 定期重启应用释放内存

### 提升响应速度
1. 将常用目录部署在 SSD 上
2. 避免部署网络驱动器或移动硬盘上的目录
3. 关闭不必要的后台程序

### 优化局域网访问
1. 使用有线网络连接（比 WiFi 更稳定）
2. 确保路由器性能良好
3. 避免同时传输大文件

## 故障排除

### 应用启动问题

**问题**: 双击 EXE 后没有反应
- 检查任务管理器中是否已有进程在运行
- 以管理员身份运行
- 检查杀毒软件是否拦截

**问题**: 启动后立即崩溃
- 检查 Windows 11 版本（需要 21H2 或更高）
- 检查 WebView2 是否已安装
- 查看日志文件：`%APPDATA%\prism-local-server\logs\`

### 服务启动问题

**问题**: 提示"找不到入口文件"
- 确认目录中包含 `index.html` 文件
- 或在设置中修改默认入口文件名
- 检查文件名大小写（Windows 不区分，但建议统一）

**问题**: 提示"端口被占用"
- 应用会自动递增端口，如果仍失败说明 8888-8988 都被占用
- 使用 `netstat -ano | findstr :8888` 查看占用情况
- 手动指定其他端口（如 9000）

**问题**: 提示"目录已被使用"
- 同一目录只能启动一个服务
- 先停止已有服务，或选择其他目录

### 访问问题

**问题**: 本地访问正常，局域网无法访问
- 检查防火墙设置（见上文"局域网访问"章节）
- 确认设备在同一局域网
- 尝试 ping 本机 IP 测试连通性

**问题**: 页面显示但资源加载失败（CSS/JS/图片）
- 检查资源路径是否使用相对路径
- 避免使用绝对路径（如 `/css/style.css`）
- 推荐使用相对路径（如 `css/style.css` 或 `./css/style.css`）

**问题**: 视频无法拖拽播放
- 确认视频格式为 MP4/WebM
- 检查浏览器是否支持该视频编码
- 使用浏览器开发者工具检查是否返回 206 状态码

### 性能问题

**问题**: 内存占用过高
- 正常情况：空闲 ~35MB，运行 5 个服务 ~120MB
- 如果超过 200MB，尝试重启应用
- 检查是否有内存泄漏（长时间运行后）

**问题**: UI 响应慢或卡顿
- 检查 CPU 占用情况
- 关闭不必要的服务实例
- 检查系统资源是否充足
